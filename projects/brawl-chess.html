📚 Documentation Complète : Brawl Chess

Publié le 09 mai 2025

Cette documentation couvre chaque aspect du projet Brawl Chess, un jeu tactique tour par tour développé avec un client Love2D (Lua) et un serveur Java. Elle détaille le code Lua, le serveur Java, et l’architecture globale pour faciliter la maintenance et la compréhension, même des parties les plus simples.

Aperçu Général 🌍

Brawl Chess est un jeu où deux camps (joueur et ennemi) s’affrontent sur un plateau 8x8. Les pions (Sniper, Tourelle, etc.) ont des mouvements et actions uniques. Le client Love2D gère l’interface et les interactions, tandis que le serveur Java valide les règles et maintient l’état du jeu.

  • Client (Lua): Affiche le plateau, gère les clics, envoie des requêtes TCP au serveur.
  • Serveur (Java): Charge les pions depuis pions.txt, valide les mouvements et actions, gère les tours.
  • Communication: Le client envoie des JSON (ex. {type="move", piece={...}}) au serveur, qui répond avec des JSON (ex. {success=true}).

Exemple de flux : Le joueur clique pour déplacer un Sniper → Lua envoie une requête → Java valide via PieceRules → Le serveur répond → Lua met à jour le plateau.

Variables Globales

Nom Description Détails/Exemple
board Tableau 2D pour le plateau 8x8. Stocke les pions ou nil. Ex: board[1][1] = {name="Sniper", hp=3}.
boardSize Taille du plateau (8). Définit les dimensions. Ex: for i = 1, boardSize do.
tileSize Taille d’une case (50 pixels). Pour dessiner et convertir les clics. Ex: boardCol = math.floor((x - boardX) / tileSize) + 1.
boardX, boardY Position du plateau à l’écran. Coordonnées en pixels pour centrer le plateau. Ex: boardX = 100.
playerPieces Liste des pions du joueur. Contient name, x, y, hp. Ex: {name="Sniper", x=1, y=1, hp=3}.
enemyPieces Liste des pions ennemis. Similaire à playerPieces. Ex: {name="Soldat", x=1, y=8, hp=2}.
selectedPiece Pion sélectionné. nil ou infos du pion. Ex: selectedPiece = {name="Sniper", x=1, y=1}.
actionMode Mode d’action actif. Ex: "attack" pour Sniper, nil sinon. Ex: actionMode = "attack".
currentTurn Tour actuel ("player"/"enemy"). Bloque les actions. Ex: if currentTurn ~= "player" then errorMessage = "Ce n'est pas votre tour !" end.
enemyTurnTimer Timer pour le tour ennemi (1s). Décompte pour repasser au joueur. Ex: enemyTurnTimer = 1.
enemyTurnDuration Durée fixe du tour ennemi (1s). Constante pour le timer. Ex: enemyTurnDuration = 1.
errorMessage Message d’erreur affiché. Ex: "Déplacement invalide". Ex: errorMessage = response.error.
client Objet de connexion TCP. Gère la communication avec le serveur. Ex: client = socket.tcp().

Fonctions Principales

Nom Description Détails/Exemple
combat.load() Initialise le plateau vide. Crée un tableau 8x8 avec nil. Ex: 
for i = 1, boardSize do board[i] = {} end
combat.enterCombat() Connecte au serveur et charge les pions. Envoie une requête vide, remplit playerPieces et enemyPieces. Ex:

combat.sendMove(piece, targetX, targetY) Envoie une requête de déplacement. JSON avec type="move", met à jour si success. Ex: 
local request = { type = "move", piece = {...} }
combat.sendAction(piece, action, targetX, targetY) Envoie une requête d’action. Gère attack, shield. Ex: 
if response.success then targetPiece.hp = targetPiece.hp - 1 end

client:send(json.encode(request) .. "\n")
combat.update(dt) Mise à jour en temps réel. Décompte enemyTurnTimer. Ex:

combat.draw() Dessine le plateau et les pions. Utilise love.graphics. Ex: 
love.graphics.rectangle("fill", ...)
combat.mousepressed(x, y, button) Gère les clics. Clic gauche : déplacer, clic droit : action. Ex: 
if button == 1 then combat.sendMove(...)
combat.mousemoved(x, y) Suit la souris pour surligner les cases. Mise à jour de la case active. Ex: 
activeCol = math.floor((x - boardX) / tileSize) + 1

if key == "escape" then love.event.quit() end

Règles des Pions 🎲

Les règles sont centralisées dans PieceRules.java. Voici un résumé :

  • Tourelle: Déplacement unique n’importe où, puis bloqué.
  • Sniper: Bouge 1 case (haut/bas/gauche/droite), action attack (1 dégât à distance).
  • Bouclier: Bouge 1 case, action shield (ajoute 1 bouclier à un allié).
  • Kamikaze: Bouge 1 case, action attack (1 dégât à 1 case, se détruit).
  • Mur: Bouge 1 case, action deploy (crée un mur adjacent).
  • Soldat: Bouge 1 case vers l’avant (y+1).

⚔️ Lancement de Brawl Chess : Client Love2D et Serveur Java

Publié le 01 mai 2025

Le projet Brawl Chess a débuté avec la mise en place d’un client Love2D et d’un serveur Java pour un jeu de combat tactique. L’objectif était de créer une base solide pour un système tour par tour avec des pions personnalisés.

🎮 Côté client (Love2D)

Le client, développé avec Lua via Love2D, inclut une interface simple pour afficher un plateau de jeu 8x8 et gérer les interactions. Voici un extrait de combat.lua pour initialiser le plateau : 

function combat.load()
                    for i = 1, boardSize do
                        board[i] = {}
                        for j = 1, boardSize do
                            board[i][j] = nil
                        end
                    end
                end

L’interface comprend :

  • Un plateau 8x8 avec des cases alternées blanc/gris
  • Un système de sélection au clic pour déplacer les pions
  • Un affichage des stats (HP, bouclier) pour chaque pion
🧠 Côté serveur (Java)

Le serveur, écrit en Java, gère l’état du jeu et les requêtes TCP. Il initialise un plateau avec des pions depuis un fichier pions.txt et valide les mouvements. 

BoardInitializer.initializeBoard(board, gameState);
🔭 Ce que ça ouvre
  • Base pour un jeu tour par tour multijoueur
  • Possibilité d’ajouter des règles complexes pour les pions
  • Évolution vers une gestion centralisée des données

if (pieceName.equals("Sniper") && action.equals("attack")) {

function combat.mousepressed(x, y, button)

🔄 Système de Tours pour Brawl Chess

Publié le 04 mai 2025

Nous avons implémenté un système de tours alternés entre joueur et ennemi, avec une transition visible pour améliorer l’expérience.

🧠 Côté serveur (Java)

Une nouvelle classe TurnManager.java gère les tours, et GameServer.java valide les actions selon le tour actuel : 

public class TurnManager {
                    private String currentTurn = "player";
                
                    public void switchTurn() {
                        currentTurn = currentTurn.equals("player") ? "enemy" : "player";
                    }
                }

L’ennemi passe automatiquement son tour pour simplifier l’IA.

🎮 Côté client (Love2D)

Le client affiche le tour actuel et bloque les actions pendant le tour de l’ennemi avec un timer dans combat.lua : 

function combat.update(dt)
                    if currentTurn == "enemy" and enemyTurnTimer > 0 then
                        enemyTurnTimer = enemyTurnTimer - dt
                        if enemyTurnTimer <= 0 then
                            combat.sendEndEnemyTurn()
                        end
                    end
                end
🔭 Ce que ça ouvre
  • Structure claire pour un jeu multijoueur
  • Possibilité d’ajouter une IA ennemie active
  • Amélioration visuelle des transitions

📋 Centralisation des Règles dans Brawl Chess

Publié le 06 mai 2025

Pour faciliter les modifications, nous avons centralisé les règles de mouvements et actions dans un nouveau fichier PieceRules.java.

🔧 Côté serveur (Java)

PieceRules.java contient les validations des déplacements, comme pour le Sniper : 

if (pieceName.equals("Sniper")) {
                    if (piece.getBoolean("hasUsedAction")) {
                        return false;
                    }
                    int dx = Math.abs(targetX - currentX);
                    int dy = Math.abs(targetY - currentY);
                    return (dx == 1 && dy == 0) || (dx == 0 && dy == 1);
                }

Les actions (ex. attaque du Kamikaze) sont aussi gérées ici, rendant le code modulaire.

🎮 Côté client (Love2D)

Aucune modification côté client, car les règles sont appliquées par le serveur.

🔭 Ce que ça ouvre
  • Facilité à ajouter de nouveaux pions
  • Maintenance simplifiée des règles
  • Préparation pour une IA ou des variantes

🎨 Ajout des Sprites PNG pour les Pions dans Brawl Chess

Publié le 11 mai 2025

Nous avons enrichi l’expérience visuelle en ajoutant des sprites PNG uniques pour chaque pion, différenciant les alliés et les ennemis.

🔧 Côté client (Love2D)

Les fichiers pieces.lua et render.lua ont été mis à jour pour intégrer les sprites :

  • pieces.lua charge les images avec pieces.loadSprites() et assigne un sprite à chaque pion via pieces.assignSprite() selon le nommage blue_sprite_1.png à blue_sprite_14.png pour les alliés, et red_sprite_1.png à red_sprite_14.png pour les ennemis.
  • render.lua remplace les rectangles colorés par les sprites, avec un redimensionnement automatique pour s’adapter à la taille des tuiles.

Dans combat.lua, les sprites sont chargés au démarrage et assignés après la récupération des données du serveur : 

function combat.load()
                    combat.pieces.loadSprites()
                    -- ...
                end

                function combat.enterCombat()
                    -- ... (récupération des pions)
                    for _, piece in ipairs(combat.playerPieces) do
                        combat.pieces.assignSprite(piece, false)
                    end
                    for _, piece in ipairs(combat.enemyPieces) do
                        combat.pieces.assignSprite(piece, true)
                    end
                end
🎮 Impact sur le jeu

Les pions sont maintenant représentés par des images personnalisées, améliorant l’esthétique et la différenciation entre alliés (bleus) et ennemis (rouges).

🔭 Ce que ça ouvre
  • Possibilité d’ajouter des animations pour les pions
  • Personnalisation visuelle selon les joueurs
  • Préparation pour des effets visuels (ex. explosion du Kamikaze)
